# 불공정 약관 분석기 - 사용 가이드

## 빠른 시작

1. 개발 서버가 실행 중입니다 (http://localhost:5173)
2. 브라우저에서 접속하여 바로 테스트 가능합니다

## 기능 테스트 방법

### 1. 샘플 데이터로 테스트하기

가장 빠른 방법:
1. 페이지 접속
2. "샘플 약관 불러오기" 버튼 클릭
3. "약관 분석하기" 버튼 클릭
4. 1.5초 후 분석 결과 확인

### 2. 직접 입력 테스트

1. "직접 입력" 탭 선택 (기본)
2. 텍스트 영역에 약관 내용 입력
3. "약관 분석하기" 버튼 클릭
4. 결과 확인

### 3. 파일 업로드 테스트

1. "파일 업로드" 탭 선택
2. "파일 선택" 버튼 클릭
3. TXT 파일 선택 (현재 지원)
4. "약관 분석하기" 버튼 클릭
5. 결과 확인

## 분석 결과 구성

### 약관 분석 요약
- 전체 조항 수
- 불공정 조항 수
- 위험도 (높음/중간/낮음)

### 불공정 약관 상세
각 불공정 조항마다:
- 조항 번호
- 원문 텍스트 (연한 빨강 배경)
- 문제점 유형
- 심각도
- 상세 설명
- 관련 법령

### 개선 권고사항
- 수정이 필요한 항목들의 목록

## 현재 상태 (개발 모드)

- API: 더미 데이터 사용 중
- 분석 시간: 약 1.5초 (실제 네트워크 딜레이 시뮬레이션)
- 파일 업로드: 클라이언트에서 텍스트 읽기

## API 연결 시 변경사항

`/src/api/termsAnalysis.js` 파일에서 주석 처리된 부분을 활성화하면 됩니다:

1. 더미 데이터 Promise 제거
2. fetch API 코드 주석 해제
3. 엔드포인트 URL 설정

## 스타일 커스터마이징

CSS 파일은 `/src/styles/` 디렉토리에 구조화되어 있습니다:

- `/styles/common/` - 헤더, 푸터 공통 스타일
- `/styles/home/` - 홈 페이지 컴포넌트 스타일

색상 변경 예시:
```css
/* 주요 버튼 색상 변경 */
.terms-input-button {
  background-color: #1f2937; /* 여기를 원하는 색으로 변경 */
}
```

## 트러블슈팅

### 서버가 시작되지 않을 때
```bash
npm install
npm run dev
```

### 빌드 오류
```bash
npm run build
```

### 포트 충돌
Vite는 기본적으로 5173 포트를 사용합니다.
`vite.config.js`에서 포트 변경 가능:
```javascript
export default {
  server: {
    port: 3000
  }
}
```
